TAKAMI. Setup

Configuración inicial de Supabase — ejecuta esto una sola vez
⚠️ Este archivo crea todas las tablas, políticas y datos iniciales. Ábrelo en tu navegador después de subir a Netlify, o directamente desde tu computador.
1
Crear tablas en Supabase
✓

Ve a Supabase → SQL Editor → New query, pega el siguiente SQL y haz clic en Run:

-- TAKAMI: Crear todas las tablas -- 1. Tabla de usuarios (perfil extendido) create table if not exists usuarios ( id uuid primary key references auth.users(id) on delete cascade, nombre text not null, rol text not null check (rol in ('admin','conductor','cliente')), email text, camion_id uuid, camion_nombre text, created_at timestamptz default now() ); -- 2. Tabla de camiones create table if not exists camiones ( id uuid primary key default gen_random_uuid(), nombre text not null, conductor text, placa text, color text default '#00c8e8', estado text default 'en_ruta', temperatura numeric default -18, progreso integer default 0, parada_actual integer default 0, paradas_completadas integer[] default '{}', paradas jsonb default '[]', conductor_id uuid, lat numeric, lng numeric, ultima_actualizacion timestamptz, ultima_entrega timestamptz, created_at timestamptz default now() ); -- 3. Tabla de entregas create table if not exists entregas ( id uuid primary key default gen_random_uuid(), numero_orden text, receptor text, cedula text, observaciones text, temperatura numeric, fotos text[] default '{}', camion text, camion_id uuid, conductor text, punto_entrega text, hora text, estado text default 'completado', cliente_id uuid, created_at timestamptz default now() ); -- 4. Tabla de alertas create table if not exists alertas ( id uuid primary key default gen_random_uuid(), tipo text default 'admin', titulo text, mensaje text, camion_id uuid, leida boolean default false, created_at timestamptz default now() ); -- 5. Habilitar Realtime en todas las tablas alter publication supabase_realtime add table camiones; alter publication supabase_realtime add table entregas; alter publication supabase_realtime add table alertas; -- 6. Políticas de seguridad (Row Level Security) alter table usuarios enable row level security; alter table camiones enable row level security; alter table entregas enable row level security; alter table alertas enable row level security; -- Políticas permisivas para usuarios autenticados create policy "usuarios_select" on usuarios for select using (auth.role() = 'authenticated'); create policy "usuarios_insert" on usuarios for insert with check (auth.role() = 'authenticated'); create policy "usuarios_update" on usuarios for update using (auth.role() = 'authenticated'); create policy "camiones_all" on camiones for all using (auth.role() = 'authenticated'); create policy "entregas_all" on entregas for all using (auth.role() = 'authenticated'); create policy "alertas_all" on alertas for all using (auth.role() = 'authenticated'); -- 7. Bucket de Storage para fotos insert into storage.buckets (id, name, public) values ('fotos-entregas', 'fotos-entregas', true) on conflict (id) do nothing; create policy "fotos_upload" on storage.objects for insert with check (bucket_id = 'fotos-entregas' and auth.role() = 'authenticated'); create policy "fotos_read" on storage.objects for select using (bucket_id = 'fotos-entregas');

Una vez ejecutado sin errores, continúa al paso 2.
